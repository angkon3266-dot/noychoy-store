# Channels & CRM — what the platform can actually do

Every capability below exists in the admin panel today. AI planning campaigns
should propose only these levers (plus FB ads).

## Owned channels
- **Web push** (Admin → Notifications → "Send a web push"): browser pop-ups
  even when the site is closed; supports title, message, image, two action
  buttons, and an attached coupon. Reaches subscribed devices only (members
  opt in via the bell; iPhone requires Add to Home Screen). Composer reports
  exactly how many devices were queued.
- **Member bell + notifications page**: every broadcast also lands in-app.
- **SMS** (paid, via gateway): optional checkbox on the same composer, and
  automated templates (order placed, delivery updates, password reset).
  Every SMS costs money — reserve for high-intent moments.
- **Email**: order invoice with signed "view order" link; more flows possible.

## Automations already running
- **Win-back**: customers with no order in N days (default 60) get an
  automatic offer + notification, with a cooldown (default 30 days).
- **Back-in-stock & price-drop push** for watchers (price drop needs ≥5% and
  respects a 24h cooldown).
- **New-arrivals batch announcement** and pre-order early-access pushes.
- **Abandoned-cart capture**: checkout leads are recorded and marked
  recovered when they order.

## Segmentation
- **RFM auto-segments** (recency/frequency/monetary) refresh automatically —
  champions, loyal, at-risk, hibernating, etc.
- **Custom segments** with rules; any segment can be targeted by push/SMS and
  linked to a personalized offer.
- **Campaign analytics** attribute orders to campaigns (7-day last-touch) —
  check them before repeating a campaign.

## Cadence guardrails
- Max ~2 broadcast pushes/week to "all members" — save the rest for segments.
- Never send the same offer twice to a segment inside its cooldown.
- Every campaign: one goal, one CTA, one tracking coupon where possible.

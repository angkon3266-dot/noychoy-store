# Order issues playbook

## "Where is my order?"
1. Look up the order (admin → Orders; status syncs from Steadfast webhooks).
2. If shipped: share the courier status + expected window.
3. If still processing beyond 24h: apologize, confirm dispatch timing.
Reply template: "Apnar order #[N] ekhon [status] — [expected day] er moddhe
pouchhe jabe insha'Allah. Kono somossa hole amra janabo! 💛"

## "I got a 429 / site error while ordering"
Take the order manually (name, phone, address, items) and create it in the
admin panel (source: manual). Then report the error to the owner.

## "The price changed in my cart"
The cart re-checks live prices at checkout — the shown correction is
intentional. Apologize for the surprise, confirm the current price, offer to
place the order.

## "It arrived damaged / wrong item"
1. Ask for a photo (WhatsApp).
2. Apologize without arguing; log a note on the order.
3. Offer per [returns policy](../policies/returns.md); escalate to the owner
   with photo + order number before promising a refund.

## "Cancel my order"
If not yet handed to courier: cancel in admin (stock restocks
automatically). If already shipped: explain they can refuse delivery
(COD unpaid), and we mark it returned when it comes back.

## Always
End with the next step + when. Log every promise as an order note so the
next person (or AI) sees it.
